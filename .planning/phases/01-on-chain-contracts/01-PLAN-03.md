## Overview

Update the existing campaign type script to add TypeID for unforgeable campaign cell identity (CAMP-01) and add destruction protection logging (CAMP-02). The TypeID enables pledge lock scripts to securely reference campaign cells via type script hash.


2. Add the new error code after the existing error codes:
```rust
const ERROR_INVALID_TYPE_ID: i8 = 12;
```

3. Insert TypeID validation as the FIRST thing in `program_entry()`, right after the opening debug line `debug!("Campaign Type Script running");` and BEFORE loading the script:
```rust
    // CAMP-01: Validate TypeID — first 32 bytes of args
    // On creation: verifies hash matches blake2b(first_input.out_point || output_index)
    // On update/burn: passes (TypeID rules allow transfer and burn)
    if let Err(_) = check_type_id(0, 32) {
        debug!("TypeID validation failed");
        return ERROR_INVALID_TYPE_ID;
    }
```

The rest of the existing logic remains unchanged.
</action>

<title>Update campaign destruction case with CAMP-02 protection comment</title>

with:
```rust
        // Destruction: has input, no output

            debug!("Campaign destruction — backers protected by fail-safe refund (D-06)");
```

Per decision D-09, on-chain destruction protection is deferred. The fail-safe refund (D-06) in the pledge lock script provides the real backer protection: if the campaign cell is destroyed, the lock script defaults to refunding the backer. The updated comment documents this architectural decision.
