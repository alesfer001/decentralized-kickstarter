<title>Cross-check requirement coverage across all contracts</title>
<action>
Verify each Phase 1 requirement is addressed by checking specific code patterns:

**LOCK-02** (deadline via since): `contracts/pledge-lock/src/main.rs` contains `load_input_since` and `Since::new` and `LockValue::BlockNumber`

**LOCK-03** (cell_dep verification): `contracts/pledge-lock/src/main.rs` contains `find_campaign_in_cell_deps` which compares `load_cell_type_hash` against `campaign_type_script_hash`

**LOCK-04** (unique args per backer): `contracts/pledge-lock/src/main.rs` `PledgeLockArgs` struct has `backer_lock_hash` field at offset [40..72]

**RCPT-01** (creation alongside pledge): `contracts/receipt/src/main.rs` `validate_receipt_creation` searches outputs for pledge cell with matching backer_lock_hash

**RCPT-02** (immutable receipt data): `contracts/receipt/src/main.rs` `(true, true)` returns `ERROR_RECEIPT_MODIFICATION_NOT_ALLOWED`

**RCPT-03** (destruction validates refund): `contracts/receipt/src/main.rs` `validate_receipt_destruction` checks output to backer with `cap >= min_refund`

**MERGE-02** (pledge type allows merge): `contracts/pledge/src/main.rs` `(true, true)` routes to `validate_merge_pledge` (N->1) or `validate_partial_refund` (1->1 reduced)

**CAMP-01** (TypeID): `contracts/campaign/src/main.rs` calls `check_type_id(0, 32)`

**CAMP-02** (destruction protection): `contracts/campaign/src/main.rs` destruction case comments reference CAMP-02 and D-06

Report any requirement not covered. Do NOT modify files.
</action>
