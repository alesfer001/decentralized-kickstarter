Implement `contracts/campaign-lock/src/main.rs` following the pledge-lock pattern (lines 296-319 from pledge-lock) but simplified for campaign-lock (no backer hash, no routing, only deadline validation).

**Lock args structure (8 bytes):**
- Bytes 0-7: deadline_block (u64, little-endian)

**Program entry logic:**
1. Load script via `load_script()` and extract args
2. Parse args into CampaignLockArgs struct with deadline_block field
3. Load since field via `load_input_since(0, Source::GroupInput)`
4. Check deadline:
   - If since_raw == 0 → return ERROR_SINCE_BELOW_DEADLINE (before deadline, not allowed)
   - If since_raw != 0:
     a. Create Since object: `Since::new(since_raw)`
     b. Validate flags: `since.is_absolute()` AND `since.flags_is_valid()` → else return ERROR_INVALID_SINCE
     c. Extract lock value: `since.extract_lock_value()`
     d. Match on LockValue::BlockNumber(block):
        - If block < deadline_block → return ERROR_SINCE_BELOW_DEADLINE
        - If block >= deadline_block → return 0 (success, allow spending)
     e. Otherwise → return ERROR_INVALID_SINCE

**Error codes (per D-03 - minimal):**
- ERROR_INVALID_ARGS = 10
- ERROR_LOAD_SINCE = 11
- ERROR_INVALID_SINCE = 12
- ERROR_SINCE_BELOW_DEADLINE = 13

**CampaignLockArgs struct:**
```rust
const CAMPAIGN_LOCK_ARGS_SIZE: usize = 8;

struct CampaignLockArgs {
    deadline_block: u64,
}

impl CampaignLockArgs {
    fn from_bytes(data: &[u8]) -> Result<Self, i8> {
        if data.len() < CAMPAIGN_LOCK_ARGS_SIZE {
            return Err(ERROR_INVALID_ARGS);
        }
        let deadline_block = u64::from_le_bytes(data[0..8].try_into().unwrap());
        Ok(CampaignLockArgs { deadline_block })
    }
}
```
