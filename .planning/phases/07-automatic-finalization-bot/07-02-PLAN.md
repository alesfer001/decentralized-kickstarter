    // Create bot signer from private key

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Environment → Bot private key | BOT_PRIVATE_KEY loaded from Render secrets; never logged (V4) |
| Indexer process → Bot methods | Bot instance shared within same process; no network exposure |
| Polling loop → CKB RPC | Bot inherits HTTPS RPC connection from indexer client (V9) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-07 | Tampering | Environment variable loading | Mitigate | Load BOT_PRIVATE_KEY from process.env only (never hardcode). Render secrets dashboard provides secure storage. |
| T-07-08 | Information Disclosure | Bot initialization logs | Mitigate | Log bot address and status to stdout, but never log private key or full config. Per D-09, console.log only. |
| T-07-09 | Denial of Service | Bot initialization failure | Accept | If bot init fails, log error and continue indexer without bot. Indexer remains operational; bot is optional feature. |
| T-07-10 | Elevation of Privilege | Signer construction | Accept | Signer constructed from private key is scoped to bot operations (finalize/release/refund). No access to other indexer functions or database writes. |
