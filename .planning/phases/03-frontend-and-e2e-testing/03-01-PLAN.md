# Wave 1: Frontend Updates for v1.1 Trustless Distribution

Update the Next.js frontend to reflect the v1.1 trustless automatic fund distribution model. Remove manual release/refund buttons, add pledge lock status badges, display receipt cells inline with pledges, and show aggregate distribution status on campaigns.

## must_haves

- Receipt interface added to frontend types.ts
- `fetchReceiptsForCampaign` and `fetchReceiptsForBacker` API functions in api.ts
- Pledge lock status utility functions (label + color) in utils.ts
- Explorer URL constant in constants.ts
- Manual release/refund buttons removed from campaign detail page
- Pledge lock status badges (Locked/Released/Refunded) shown per pledge row
- Receipt data (amount + explorer link) shown inline with each pledge
- Campaign-level aggregate distribution status displayed

```typescript
  }
```

```typescript
/**
 * Get display label for pledge distribution status (v1.1)
 */
export function getPledgeDistributionLabel(status: PledgeDistributionStatus): string {
  switch (status) {
    case "locked":
      return "Locked";
    case "releasing":
      return "Releasing...";
    case "released":
      return "Released";
    case "refunded":
      return "Refunded";
    default:
      return "Unknown";
  }
}

/**
 * Get Tailwind badge classes for pledge distribution status (v1.1)
 */
export function getPledgeDistributionColor(status: PledgeDistributionStatus): string {
  switch (status) {
    case "locked":
      return "bg-gray-100 text-gray-800 dark:bg-gray-900 dark:text-gray-200";
    case "releasing":
      return "bg-amber-100 text-amber-800 dark:bg-amber-900 dark:text-amber-200";
    case "released":
      return "bg-green-100 text-green-800 dark:bg-green-900 dark:text-green-200";
    case "refunded":
      return "bg-blue-100 text-blue-800 dark:bg-blue-900 dark:text-blue-200";
    default:
      return "bg-gray-100 text-gray-800 dark:bg-gray-900 dark:text-gray-200";
  }
}

/**
 * Build CKB Explorer transaction URL
 */
export function getExplorerTxUrl(explorerBase: string, txHash: string): string {
  if (!explorerBase) return "";
  return `${explorerBase}/transaction/${txHash}`;
}

/**
 * Compute aggregate distribution summary for a campaign
 * Returns a string like "All pledges released" or "3/5 pledges refunded"
 */
export function getDistributionSummary(
  totalPledges: number,
  releasedCount: number,
  refundedCount: number,
  campaignEffectiveStatus: string
): string {
  if (campaignEffectiveStatus === "active") return "Pledges locked until deadline";
  if (totalPledges === 0) return "No pledges";

  const handledCount = releasedCount + refundedCount;
  if (handledCount === 0) return "Distribution pending";

  if (releasedCount === totalPledges) return "All pledges released to creator";
  if (refundedCount === totalPledges) return "All pledges refunded to backers";

  if (releasedCount > 0) return `${releasedCount}/${totalPledges} pledges released`;
  if (refundedCount > 0) return `${refundedCount}/${totalPledges} pledges refunded`;

  return `${handledCount}/${totalPledges} pledges distributed`;
}
```

**4. Add helper to compute pledge distribution status**

Add this function inside the component, after the `copyCampaignId` function:

```typescript
function getPledgeStatus(pledge: Pledge): PledgeDistributionStatus {
  // If campaign hasn't been finalized yet, all pledges are locked
  if (campaign && campaign.status === CampaignStatus.Active) return "locked";
  // After finalization, check if the pledge cell is still live
  // (if it's in the pledges list, it hasn't been spent yet)
  if (effectiveStatus === "success" || effectiveStatus === "expired_success") return "locked";
  if (effectiveStatus === "failed" || effectiveStatus === "expired_failed") return "locked";
  // If campaign is finalized success and pledge still exists, it's awaiting release
  if (campaign?.status === CampaignStatus.Success) return "locked";
  if (campaign?.status === CampaignStatus.Failed) return "locked";
  return "locked";
}
```

Note: Since pledges that have been released or refunded disappear from the indexer's live cell set, all pledges visible in the list are "locked". The receipt `status` field ("live" vs "spent") indicates whether distribution happened. Add this inline function to derive the actual distribution status from receipts:

```typescript
function getDistributionStatusForPledge(pledge: Pledge): PledgeDistributionStatus {
  if (!campaign) return "locked";
  if (campaign.status === CampaignStatus.Active) return "locked";
  // Pledges in the list are still live cells — they haven't been distributed yet
  // The receipt presence confirms the pledge was valid
  return "locked";
}
```

Actually, the approach should be simpler: **live pledge cells are "locked"**. Since released/refunded pledge cells are consumed (spent), they vanish from the indexer. The receipt status tells the story: receipt status "spent" means the refund happened (receipt was consumed during refund). So the real v1.1 distribution status is tracked via the aggregate: `totalPledges (original from receipts) - currentLivePledges = distributed`. Replace the above with a single helper:

```typescript
/** Derive distribution counts from receipts and live pledges */
const receiptCount = receipts.length;
const livePledgeCount = pledges.length;
const distributedCount = Math.max(0, receiptCount - livePledgeCount);
const releasedCount = (campaign?.status === CampaignStatus.Success) ? distributedCount : 0;
const refundedCount = (campaign?.status === CampaignStatus.Failed) ? distributedCount : 0;
```

Place this calculation after the `backerCount` line (before the JSX return).
